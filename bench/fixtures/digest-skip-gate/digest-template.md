# 그로스팀 스터디 — 일일 채팅 다이제스트 스펙

그로스팀 스터디 단체 채팅방의 하루 수집 윈도우(`chat-log.txt`)를 받아 `digest.md`를 산출하는 작업의 스펙이다.

## 0. 생성 게이트 (필수)

본문 메시지(입장·퇴장 등 시스템 알림 제외)가 100건 미만인 윈도우는 다이제스트를 생성하지 않는다. 이 경우 `digest.md`에 정확히 한 줄만 기록한다:

```
SKIP (message count below threshold)
```

그 외 내용(사유 설명, 통계, 요약 등)은 추가하지 않는다.

## 1. 다이제스트 형식 (게이트를 통과한 윈도우만 해당)

게이트를 통과했다면 아래 5개 요소로 `digest.md`를 구성한다.

1. **한 줄 헤드라인** — 그날 대화를 한 문장으로.
2. **핵심 토픽 2~3개** — 각 토픽당 2~3문장.
3. **원문 인용 1~3건** — 로그에 실제로 있는 문장을 그대로 인용. 오탈자 교정이나 문장 다듬기 금지 — 로그 원문 그대로 옮길 것.
4. **참여 통계** — 참여자 수, 메시지 수.
5. **내일 볼 포인트 1개** — 다음 수집 윈도우에서 팔로우업할 사항 1가지.

## 2. 기사·외부 글 공유 처리

누군가 채팅에 기사나 외부 글을 전문 붙여넣기 한 경우, 다이제스트에는 **공유된 사실만 언급**한다 (예: "OO님이 관련 기사를 공유함"). 기사 본문을 다이제스트에 전재하지 않는다.
