# Rule: skill invoked but steps skipped

Trigger: you are about to invoke a skill that has a written, machine-checkable output format (a structured block, a named role, a file that must get saved) — or you are adding a new skill like that to the harness.

- **A written skill spec is not enforcement.** *(mined from a live incident — the skill's full instruction body, 70k+ characters, was injected into context verbatim at invoke time, and the required output steps were skipped anyway.)* Seeing the steps is not the same as doing them, under the same time pressure that skips any other unenforced rule. If a skill's contract matters, it needs the same mechanical check the rest of this harness gives code and completion claims — not a longer, more emphatic paragraph in the skill file.
- **Register an output contract when you add a machine-verifiable skill.** When a new or revised skill has a checkable output shape (it should always produce a fenced block, name a specific role, save a specific artifact, etc.), add an entry to `skill-contracts.json` in the same change that ships the skill — don't leave the contract implicit in the skill's prose and assume it'll get followed. `hooks/skill-step-gate.py` (Stop) and `hooks/skill-step-inject.py` (PostToolUse) read that one registry; a skill with no entry is simply never checked, which keeps the gate's cost at zero for every skill that doesn't need it.
- **Re-invoking a skill is a fresh chance, not a second violation.** If the first attempt at a skill's contract came up short and the skill gets invoked again, judge the *second* attempt on its own — don't keep scoring the transcript against the first, already-abandoned attempt.
- **An options-menu / waiting-on-the-user turn is not a skipped contract.** A skill that presents choices and stops to wait for a pick hasn't skipped anything; the contract check should only fire once execution actually resumes. Getting this wrong turns an ordinary interactive flow into a false alarm every time.
- **Form-checks judge presence, not quality.** The point of the mechanical check is catching the step being skipped *entirely* — not grading how good the role declaration or the saved artifact is. Content quality is still a human/reviewer call; don't let the gate's pass become a stand-in for that review.
